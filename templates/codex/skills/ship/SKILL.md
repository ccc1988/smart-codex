# Skill: ship

## 目标
- 输出可直接用于交付/发版的材料
- 不写代码、不改实现

## 必须输出
- 改动说明（面向业务/用户）
- PR 描述（摘要/动机/范围/测试）
- 验收清单（可执行）
- 风险与回滚方案

## 工作流
1) 复述交付目标（1 句）
2) 默认先基于“当前工作目录”的 Git 状态汇总改动点与影响面（不改代码）
3) 若当前目录不是 Git 仓库或无可用信息，再询问用户仓库路径（二选一）
4) 生成 PR 描述草案
5) 生成验收清单
6) 提示风险与回滚步骤

## 输出约束
- 不新增实现、不修改代码
- 默认中文输出；代码/命令/路径原样

## 交付材料模板
- 改动说明（3–6 条，面向业务/用户）：
- PR 描述（摘要 / 动机 / 影响范围 / 验证）：
- 验收清单（≤5 条，可执行）：
- 风险与回滚（一句话）：

## 禁止事项
- 假设未验证的结果
- 省略回滚或验收
