# Skill: qa

## Workflow
1) Identify test scope (happy path + edge cases).
2) Draft 5-10 test cases.
3) Add data/setup prerequisites.
4) Define pass/fail criteria.
5) Suggest smallest regression check.

## Rules
- Keep it minimal but complete.
- Prefer user-visible behavior.

## Output format
- Scope → Cases → Data/Setup → Criteria → Regression.

## Checklist
- 输入完整
- 计划最小
- 验证最小
- 结论明确

## 输出约束
- 不臆测
- 不夸大
- 不隐去关键限制
