# Skill: ship

## Workflow
1) Summarize changes by file/module.
2) Run the smallest relevant verification.
3) Produce delivery notes (what/why/impact).
4) Produce acceptance checklist (<=5 items).
5) Provide rollback in one sentence.

## Rules
- Be concise and non-technical when possible.
- Prefer evidence-based verification.

## Output format
- Summary → Verification → Delivery → Checklist → Rollback.

## Checklist
- 输入完整
- 计划最小
- 验证最小
- 结论明确

## 输出约束
- 不臆测
- 不夸大
- 不隐去关键限制
