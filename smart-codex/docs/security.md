# 安全说明

SmartCodex 以“最小权限、最小改动”为核心。

## 原则
- 不输出密钥、token、个人路径
- 不运行破坏性命令
- 不做大规模重构

## 配置建议
- `sandbox` 使用 `workspace-write`
- `ask_for_approval` 使用 `on-request`
- MCP 仅在需要时启用

## 数据与隐私
- 不写入个人信息到配置或模板
- 不在仓库中保存历史会话
- 不上传敏感日志

## 操作安全
- 涉及删除/重置时必须确认
- 任何不确定行为先提问
- 所有输出需可验证

## 供应链风险
- MCP 依赖来自外部源
- 建议在受控环境运行
- 必要时锁定版本

## 事故响应
- 立即停止自动化执行
- 回滚到备份版本
- 记录原因与影响范围

## 变更管理
- 重要规则修改需复审
- 文档变更需同步更新
