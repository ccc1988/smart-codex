# Skill: doc

## Workflow
1) Confirm audience (developer/user/boss).
2) Draft a short outline.
3) Write sections with bullets.
4) Add "How to verify" and "FAQ" when useful.

## Style
- Clear, short, non-technical where possible.
- Use headings that match the outline.
- Avoid marketing tone.

## Output format
- Title + sections.
- Bullets first, paragraphs last.

## Checklist
- 输入完整
- 计划最小
- 验证最小
- 结论明确

## 输出约束
- 不臆测
- 不夸大
- 不隐去关键限制
